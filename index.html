25 + 5 Clock

Source: certification project

Details:

Another React project, this was to make a workout timer. It has a Session length and a break length, and runs one then the other until stopped. It was a simple yet complicated project. It runs by starting a setInterval() of my cycle() function that runs every second. The cycle function first checks if this is the first cycle, if it isn't it calculates the end time in milliseconds. Then it decrements the time by one second and checks if the timer the timer value. If the value is > 0 it continues updating and moves to the next cycle. If the value is 0 it ensures the 0 value is displayed and continues to the next cycle. On the following cycle, it stops the current interval, and sets the boolean isBreak to !isBreak and starts the next timer based on it's value.

Looking Back:

This project was challenging because I ran into a few errors. I wracked my brain to solve the issue and never really did. The '0 should > 0' didn't happen anymore simply because I started over after realizing how i was calculating the end time could be done a lot easier. Instead of using Date I could simply store it as a constant value and decrement 1000 milliseconds every second. This enable a vast simplification of my pause/resume. After hugely simplyfying my solution with this embarrassinly obvious alternative all my errors dissappeard, the clock ran perfectly in testing and finally passed all the tests. So, looking back I gained a greater understanding of some of the time based functions work. (setInterval, Date, setTimeout).

Quote Machine

Source: certification project

Details:

This was another React project for certification. The goal was to make a quote machine that randomly generated a different quote each time. I obviously succeeded, but again I had a little bit of fun with it. I used quotes from one of the strangest and funniest characters in the Elder Scrolls series, 'Maiq The Liar'. A reacurring khajit character that always has weird facts or boasts about the world of Elder Scrolls. The project uses 3 arrays of strings for the quote, author, and the game from which the quote originates. I added that last one for fun. The project works by picking a random number between 0 and the length of the arrays - 1. It then gets the data from each array and displays it using setState and {this.state.exampleValue}. To ensure that no quote displays more than once (until it reaches 0 at least) each item is removed from the arrays held by the React object, leaving the originals untouched. When it does eventually reach 0, it resets the arrays and starts again.

Looking Back:

Looking back I enjoyed this project despite it's simplicity because I decided to have fun with it. I would however change something. That is how the data is retrieved. I would have liked to try to retrieve the data from the site where I found the quotes, allowing for even more data. More importantly though, I would have liked to use this to implement quotes from other characters I enjoy, perhaps even setting a selector for all charactors, or specific ones! I looked into this, but the largest issue is that the most reliable source for these quotes didn't always have quotes and were sometimes formatted differently as it is a wiki site and thusly can be edited by anyone.

API Demo

Source: demo

Details:

This project is a demo I made to demonstrate fetching data from an API. This example uses a movie api to display movies from a database based on a selected genre. It works by using an async function (await fetch) to fetch data from the API database as a given .json object. Then from the json object it extracts the data and calculates the number of pages based on the results and displays the images of the given indexes. Ex: each page has 16 movies so the 17th movie (index 16) would be the first of page 2. The next/prev/first/last page buttons all work by simply changing the selected indexes and changing the display.

Looking Back:

I would like to change how the movies display, specifically to make them more dynamic. As of now it has a set of blocks that take in data. I would like to change it to dynamically create only those it needs, so that when the user gets to the end of the list there aren't empty blocks.
